⚙ Jobs & Tools

Trigger background operations on your library. Each job runs in the background — you can navigate away and return to check progress.

🎯 Score

Running…

Run enabled scorers (blur, aesthetic, NSFW, …) over all indexed photos and store the results in the library. Perceptual hashes are computed first (needed for clustering).

🔗 Cluster

Running…

Group near-duplicate photos using perceptual hashing (dhash). Requires scoring to have been run first so that pHashes exist. Results appear in the Clusters view.

📤 Export Best-Of

Running…

Copy the best representative from each cluster to the takeout-rater/exports/ folder. Requires clustering to have been run first. Optionally rank by a scorer metric to pick the highest-quality photo from each group.

🔑 Rehash (SHA-256)

Running…

Compute SHA-256 content hashes for indexed assets that don't have one yet. Hashes are used to detect true duplicates (byte-identical files).

🔄 Rescan library

Running…

Re-process assets through the indexing pipeline. Use Missing only to update assets that haven't been processed at the current indexer version (e.g. after an upgrade), or Full to re-index everything. Scores and cluster data are never modified.
