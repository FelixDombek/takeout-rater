⚙ Jobs & Tools

Trigger background operations on your library. Each job runs in the background — you can navigate away and return to check progress.

🔄 Rescan library

Running…

Re-process existing assets through the indexing pipeline: re-parses sidecar metadata and regenerates thumbnails. Use Missing only to update assets that haven't been processed at the current indexer version (e.g. after an upgrade, or when thumbnails failed to generate), or Full to re-index everything and regenerate all thumbnails. Scores and cluster data are never modified.

📁 Re-index library

Running…

Scan the Takeout folder and discover new photos to add to the library. Use this after adding new Takeout archives or if the initial setup index was interrupted. To fix thumbnails or metadata for already-indexed photos, use Rescan library above.

🔑 Rehash (SHA-256)

Running…

Compute SHA-256 content hashes for indexed assets that don't have one yet. Hashes are used to detect true duplicates (byte-identical files).

🔍 Compute CLIP Embeddings

Running…

Compute CLIP ViT-L/14 image embeddings for all assets. These embeddings enable semantic search — finding photos by describing their content in natural language (e.g. "sunset over the ocean", "birthday cake"). Requires the same ~900 MB CLIP model used by the aesthetic and CLIP-IQA scorers.
